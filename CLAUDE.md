# CLAUDE.md

Guidance for Claude Code (and teammates) working in this repo.

## What this is

Pure Deli's marketing website — an Icelandic deli/eatery. Content language is **Icelandic** (English only appears in technical contexts or a few not-yet-translated strings).

News content is pulled from the client's existing **WordPress** site (`https://puredeli.is`) via the public WP REST API. The ordering flow is embedded from a third-party widget (`widget.upsell.is`).

## Stack

- **React Router 7** (framework mode, SSR enabled — `react-router.config.ts`)
- **React 19** + **TypeScript**
- **Tailwind CSS v4** via `@tailwindcss/vite` (v4 uses `@theme` and `@utility` in CSS, not `tailwind.config.js`)
- **Vite**
- **Bun** as package manager/runtime (see `bun.lock`) — use `bun` commands
- Deployable to **Cloudflare Workers** (`@react-router/cloudflare` installed)

## Scripts

```bash
bun run dev         # dev server
bun run build       # production build
bun run start       # serve the production build
bun run typecheck   # react-router typegen && tsc -b
bun run lint        # ESLint
```

## Project layout

```
app/
  components/          # shared components (PascalCase.tsx)
    _shared/icons/     # reusable SVG icons (ArrowRightIcon, etc.)
    news/              # news-specific components (NewsCard)
  routes/              # file-based routes (registered in app/routes.ts)
    _layout.tsx        # wraps routes with Header + Footer
    home.tsx
    order.tsx          # /panta — embeds upsell.is order widget
    order.css          # widget overrides (split by sub-page)
    news.tsx           # /frettir — news list
    news-detail.tsx    # /frettir/:slug
    privacy-policy.tsx # /personuverndarstefna
  styles/              # scoped CSS imported from order.tsx for widget theming
  types/
    global.d.ts        # ambient types (e.g. window.upsell_widget)
    wordpress.ts       # WPPost, WPRenderedText, WPFeaturedMedia
  utils/
    cn.ts              # classnames helper (hand-rolled, no dep)
    constants.ts       # WORDPRESS_BASE_URL
    formatDate.ts      # Intl.DateTimeFormat("is-IS")
  index.css            # Tailwind import, fonts, theme tokens, .wp-content styles
  root.tsx             # HTML shell, favicons
app/routes.ts          # explicit route config (not file-convention auto-routing)
public/                # static assets (images, favicons, icons.svg)
```

Routes are **explicitly declared** in `app/routes.ts` — when adding a route, register it there.

## Conventions

### Language / translations
- User-facing copy is **Icelandic**. When adding text, write it in Icelandic.
- **URL paths are also Icelandic** — e.g. `/panta` (order), `/frettir` (news), `/personuverndarstefna` (privacy policy). Follow this convention when adding new routes.
- Common terms: Matseðill (menu), Veislur (parties), Fréttir (news), Mínar síður (my account), Panta (order), Hafðu samband (contact us), Lesa meira (read more), Sjá meira (view more), Nýjast (latest), Til baka (back).

### Styling
- **Tailwind v4** — theme tokens live in `@theme inline` in `app/index.css` (colors: `olive`, `sage`, `cream`, `footer`, `header`, `hero`, `off-white`, `accent`, `dark`; fonts: `font-heading`, `font-body`).
- **Responsive breakpoint**: the project normalizes to `lg:` (1024px). Prefer `lg:` over `md:`.
- Headings use `font-heading` (Moret serif). Body uses `font-body` (General Sans).
- **Heading scale pattern** (see existing components for reference):
  - Hero h1: 42/62
  - Section h2: 38/52 or 38/42
  - Card h3: 20/22 or 24/28
- `.wp-content` class in `app/index.css` styles HTML rendered via `dangerouslySetInnerHTML` (from WordPress or the privacy-policy page). Covers h1–h6, p, a, ul/ol, li, figures, and `.wp-block-gallery`. Bullet color is `#d7d7d7` via `::marker`.

### Data fetching
- WordPress list/detail fetches happen in route **loaders** (server-side SSR). See `app/routes/home.tsx`, `news.tsx`, `news-detail.tsx`.
- List endpoints (`news.tsx`, `home.tsx`) wrap the fetch in try/catch and return an empty array on failure — the page should **not crash** if WordPress is unreachable.
- The detail route returns a `404 Response` if the slug doesn't match a post.
- `WORDPRESS_BASE_URL` is `https://puredeli.is/wp-json/wp`. Always append `?_embed` when you need featured media.
- Post dates use `formatDate()` from `app/utils/formatDate.ts` — localized to `is-IS` (outputs e.g. "23. jún. 2025").

### TypeScript
- `erasableSyntaxOnly: true` is set in `tsconfig.app.json`. **Do not use `enum`, `const enum`, or `namespace` with runtime code.** This also means the `wp-types` npm package cannot be used (it ships raw `.ts` with `const enum`s). Define minimal types locally in `app/types/wordpress.ts` instead.
- `verbatimModuleSyntax: true` is set — import types with `import type { ... }`.
- Route types come from `./+types/<route-name>` (generated by `react-router typegen`). Use `Route.LoaderArgs`, `Route.ComponentProps`, `Route.MetaArgs`.

### Components
- Presentational components take data via props (e.g. `<LatestNews items={...} />`). Keep route data fetching in loaders, not inside components.
- Icon components live in `app/components/_shared/icons/` and accept `React.SVGProps<SVGSVGElement>`.
- WordPress titles contain HTML entities — render them with `dangerouslySetInnerHTML={{ __html: title }}`.

### `cn` helper
- `app/utils/cn.ts` is a hand-rolled `classnames` clone. There are two pre-existing strict-TS errors in this file (`for...in` over a union type) that are unrelated to new work — don't try to "fix" them as part of other tasks unless asked.

## Gotchas / things that already bit us

- **`erasableSyntaxOnly` vs third-party types.** See above — it blocks packages that ship `const enum` (e.g. `wp-types`). Prefer hand-written minimal types.
- **Favicon caching.** Browsers heavily cache favicons. After changing favicon markup, hard reload or use incognito. SVG favicons also take precedence over PNGs in modern browsers.
- **`STORE_ID` in `app/routes/order.tsx`.** There's a commented-out VISKA store ID used for local dev. **Commits must keep the Pure Deli store ID active** (`6970d21403e86bbbf6a6d40c`). If you see the diff toggling to the VISKA ID, revert that line before committing.
- **News detail URLs use the WP slug**, not the full WP permalink. The loader maps `post.slug` → `/frettir/:slug`.
- **The order widget** is loaded via a `<script>` tag in `app/routes/order.tsx` that pushes to `window.upsell_widget.q`. The ambient type for `window.upsell_widget` lives in `app/types/global.d.ts`. Widget-internal class names are targeted by scoped CSS in `app/styles/*.css` (per sub-page: orderMethodPage, checkoutPage, myAccountPage, orderDetailPage).
- **Link to `/panta` with `<a>`, not `<Link>`.** The order page embeds a third-party widget that needs a full page load to initialize cleanly. SPA navigation via `<Link>` breaks the widget. `Header.tsx` has a `LinkWrapper` that checks `href.includes("/panta")` and renders an `<a>` instead. Follow the same convention anywhere you link to `/panta`.
- **Tailwind v4 theme** — there is no `tailwind.config.js`. Add tokens in `@theme inline` within `app/index.css`.

## Common tasks

- **Add a route:** create the file under `app/routes/`, then add `route("path", "routes/<file>.tsx")` to `app/routes.ts`.
- **Add a WP-content style:** edit `.wp-content` rules in `app/index.css`. The class is applied in `news-detail.tsx` and `privacy-policy.tsx`.
- **Add a translation:** edit the component directly — there's no i18n framework.
- **Run typecheck:** `bun run typecheck`. Ignore the two pre-existing `cn.ts` errors.
